Security Considerations

The Kuali development team recognizes the importance of security. Solutions for authentication, authorization, and encrypting sensitive data have been provided. It will be up to each implementing institution to provide the necessary setup, configuration, and possibly alternative implementations for these important functions.

Central Authentication Service - The version of the Central Authentication Service (CAS) shipped with KFS 1.0 currently does not require passwords. Implementing institutions will need to configure CAS to require passwords in order to activate secure authentication to the application

Encryption Service - KFS 1.0 uses the Demonstration Encryption Framework to secure and encrypt sensitive data. Implementing institutions will need to decide whether to use the delivered encryption framework or another encryption framework of their choosing. A unique key will need to be established as well

User Service - KFS 1.0 is delivered with the Kuali User Service for looking up users and determining authentication credentials, workgroup membership, and authorization permissions. Implementing institutions may choose to plug the Kuali User Service into a centralized source for user data such as an LDAP directory or institutional user service. This is the preferred approach. Otherwise, user data can be created and maintained within KFS as a stand-alone application. In addition, unique key stores must be established to support the encryption framework protecting user identifiable data.

Web Service - KFS 1.0 utilized Kuali Enterprise Workflow (KEW) for the routing and approval of financial transaction documents and for managing workgroup memberships. Communications between KFS and KEW happen via Web Services and XML. A unique set of key stores must be established for KEW in order to support the encryption framework protecting document data being processed by the workflow engine.

Downloads

Before downloading, you must acknowledge that you have read the licensing and implementation requirements sections above.

Dowload the distribution of your choice.

  1. Dowload the distribution of your choice.
    Tomcat, KFS war, CAS war, KEW war
    Tomcat, contents of the KFS CVS module, CAS war, KEW war
    Contents of both the binary and source distributions
  2. Extract the zip file that you downloaded to the folder of your choice. We'll assume that you extracted the zip into /java/kfs in subsequent instructions.
  3. Import the Eclipse project in the Source or Binary & Source distribution into Eclipse.
    Note: This step is optional. We use Eclipse for development, so we have included the Eclipse project files in the distributions to allow you to work in the same IDE if you choose to.
    • Select "Import" from the "File" menu in Eclipse.
    • Select "Existing Projects into Workspace" and click "Next >".
    • Navigate to /java/kfs, select the distribution folder, and click "Choose".
    • Click "Finish".
  4. Obtain and set up Ant, if you did not do the optional step above and do not already have it.
  5. Set up your database using these instructions. You will want to use the KULDEMO export.
  6. Execute the help target of /java/kfs/distribution/build-packaged.xml and follow those instructions.
